git checkout feature/name git switch -c feature/name git merge feature/name

3. Отправка в удалённый репозиторий

git remote add origin <url>
git push -u origin main
git push origin feature/name

4. Полезные приёмы

  • Посмотреть историю: git log --oneline --graph --all
  • Откат изменений в рабочем каталоге: git checkout -- <file>
  • Откат коммита (создать новый коммит): git revert <hash>
  • Отмена локального коммита (без сохранения): git reset --hard <hash>

5. Алгоритм создания Pull Request

  1. Создать ветку от master/main: git checkout -b feature/имя
  2. Сделать коммиты: git add + git commit
  3. Запушить ветку: git push origin feature/имя
  4. Открыть Pull Request на GitHub и выбрать шаблон PR
  5. Исправить комментарии ревью и смержить в main
